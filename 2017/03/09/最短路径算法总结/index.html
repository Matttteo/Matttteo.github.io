最短路径算法总结

Posted on 2017-03-09   |  

一、最短路径问题的定义

对于一个图，任意取两个点做起点和终点，可以规划出n条起点到终点的路线，不兜圈子不重复经过某一点的路线叫做一个路径（path），如果图中的边有权重，那么这些路径也就有了长度。

最短路径是从起点到终点长度最短的路径，当图没有负边的时候，这个问题是一个NP 完全的问题，最长路径问题类似，当没有正边是NP完全问题。

最短路径有一个很好的性质：

每一条最短路径，都是由其他最短路径延伸得到，也就是说对一个最短路径进行首尾切割得到的中间一段仍然是最短路径。

从一个起点出发，到图中所有点的最短路径构成一个最短路径树（可能不止一个）。

一个起点和终点间所有最短路径组成一个最短路径图，只有一个，他是一个有向無環圖（Directed Acyclic Graph, DAG）。

最短路径算法中一个经常用到的概念是松弛（relaxation），如下图所示[1]。

最短路径问题可以分为三类：

  1. Point to Point shorest path

  2. Single source shorest path

  3. All pairs shorest path

二、单源最短路径问题

1. Label Setting 算法

所谓label就是从起点到所有终点的距离值，label setting就是算法每次外循环set一个终点的值作为最后的结果。要怎么做呢？

定义w[a][b]为a顶点到b顶点的距离，也就是边的权重，定义d[a]为起点到a的距离，除了起点本身，其他都设为0。

  1. 将起点加到最短路径树上

  2. 重复V-1次，直到所有顶点加到最短路径树上

    a. 找一个不在最短路径树并且离起点最近的点，方法是找一个已经在最短路径上的点a以及一个不在最短路径点上的b，使得w[a][b] + d[a]最小。

    b. 将b的最短路径加到d[b]上

    c. 将b点连同ab边加到最短路径树上。

这里需要证明的一点是，所有与目前最短路径树连接的，与起点最近的一个顶点可以被加入到最短路径树上。

简单说一下我的理解：在每次循环开始时，我们已经有一个最短路径树T。按照上面算法第2步找到的顶点p一定是所有与这个树连接的顶点集合P中距离起点s最短的点，那么所有P中其他点到起点的距离都会长于p，因为没有负边，那么所有不在P和T中顶点到起点的距离更不可能比p到起点的距离短，所以这一步不会改变T是一个最短路径树的事实。

我们直接实现Label Setting算法，假如顶点数量是n，时间复杂度为O(n^3)。之后的Dijkstra算法对这个问题有改进。

2. Dijkstra算法

在Label Setting算法中，我们要用穷举的方式找到一个最近的点。实际上我们做了很多重复的计算，我们可能对某些顶点到起点的距离计算了很多遍。自然的，我们考虑用一个表来保存一些目前已经知道的“最短”距离，这个距离只是当前最短的，当对最短路径树加入新的顶点后，我们需要对这个表进行更新，也就是做一些“松弛”。

进一步，如何找到目前距离向量中最短的那一个我们可以用一个优先队列来处理，具体说来就是我们把上面程序里的dists数组放到一个优先队列中，这样就可以在logn时间内取最小值了。在具体实现中，为了避免对队列里的元素进行修改，可以跳过修改直接把新的元素放到队列里，并且在取最小值时把那些已经访问过的元素跳过，这样虽然增加了点时间，但是减少了很多繁琐的对队列元素的修改删除操作。
