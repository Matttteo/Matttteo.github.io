一、二叉树的定义

所谓二叉树，就是每个树节点都有两个孩子的树，两个孩子分别叫左孩子和右孩子，二叉树这种数据结构可以用于二叉查找树，有关二叉树的性质见wiki.

二、二叉树的遍历

二叉树的遍历可以分为前序，中序，后序三种，由于每个二叉树的孩子本身也是一个二叉树，所以这种遍历是一种递归的形式，那么我们自然可以把代码写成递归，比如先序遍历：

而我们知道，所谓递归调用，在计算机中是通过内存空间里的栈来实现的。栈是一个先入后出的线性数据结构，如果我们想实现函数1调用函数2的结果，我们只要把函数2的帧放到1的上面，由于指令总是取栈顶的栈进行处理，当处理完毕后把栈顶帧pop掉，那么我们就保证了只有函数2执行完了我们的函数1才继续执行，通过这样一个简单的机制，函数的相互调用就得以有序进行了。

而递归调用是调用自己，和调用其它函数一样，递归调用也要在当前帧上放一个子问题的帧，当子问题处理完了再来处理当前帧。所以，递归调用并没有什么新鲜的东西，无法是一种函数调用而已。

问题是我们怎么知道我们处理当前的问题1的时候是应该调用子问题2还是执行后面的代码或者直接返回了呢？这是通过返回地址实现的，所谓返回地址，是指当某个函数执行完了后，CPU去读取的下一条指令的地址，也就是说每个函数1在调用其他函数2的时候要把返回地址也放到自己的帧上面，这样当2结束的时候，1就知道从哪里继续了。

上面这个算法，非常简单有效的解决了函数调用的问题，也是我们解决递归问题的核心。

再多说两句，程序被编译成二进制代码后，背后的实质是指令被存储起来之后顺序执行。指令要操纵的是什么呢，寄存器和内存。寄存器都有编号，内存在操作系统的地址转换机制保障下是一个很大的数组。一个最简单的a+b运算，我们可以把a,b放到两个寄存器，然后调用加法指令把两个寄存器内容相加保存到一个新的地址上去。computer毕竟不只是计算器，在计算机里函数会互相调用，举个函数调用的例子：

这个例子里，main调用了sum，如果说不使用栈来搞定这个函数执行会如何呢？我们需要把a,b,c分别放在内存或寄存器中，而且我们必须在编译的时候就确定他们的地址。在这个例子中，我们还是可以处理的，无法是多了一个变量而已，但是如果调用的函数特别多，变量也特别多的时候，我们必须一开始就把所有函数中的变量地址都提取设置好地址，即使他们还没被执行到，或者说他们已经被执行完了。并且有许多变量在没有执行的时候，根本不知道需要多少变量，可能还需要根据用户的输入来临时生成变量。这种情况下，需要执行代码的时候动态的分配变量。使用栈我们就可以通过函数在栈中的相对位置来分配变量的地址了。

当然在C语言中，我们也是可以根据输入动态分配变量的，但是那是通过指针的方式，使用的是动态内存，也就是堆中的内存。在C语言中我们知道，变量要先声明才能用，一个数组要首先声明他的大小才能用，一个结构体，一个复杂的数据结构也都是这样的。如果我们不用栈，而是在编译的时候就把变量的绝对地址指定，那么会很麻烦的。

如果一定要实现，我们设想一个表，每个函数对应一个表项，每个表项里面是这个函数需要的所有变量的地址。当编译的时候，我们生成这个表，然后我们确实不需要栈了，我们一条条执行指令，遇到一个变量就从表里面读这个变量的地址。当然在真实的计算机里，不是这样做的。

回到树的遍历问题上，对于递归版本，其实很简单，就是用内存中的栈来实现递归，并且每个函数在调用子函数的时候，不会直接把当前的栈帧给消除，而是利用返回地址来保证顺序执行（如在先序遍历的代码里，当前函数要调用自己处理左孩子的时候，帧里面保持下一行代码的地址，而下一次保存return行的代码）。这个函数的栈帧不止保存一个参数，其实他还会保持一些其他的东西，所以一个栈帧比一个元素大多了。并且每创建一个帧，要很多的代码，这也会使得执行很慢，有时候还会出现递归层次太深导致stack overflow的问题，所以有时候我们考虑不用递归解决问题。

另外一个方法是添加一个标记位，在取出栈顶元素的时候看看标记为，如果右子树没进栈过，就不弹出，并且修改标记。如果标记位显示右子树已经进栈过了，那么输出并弹出。这样会增加空间，但是已经不错了。

深入理解快速排序

Posted on 2017-03-09   |  

快排的思想是每次找一个元素把数组大致非为大于这个元素和小于这个元素（可以等于）的两个部分，然后在这两个部分递归的进行排序。

那么我们怎么找到一个元素在整个数组中的位置呢，最简单的方法，重新开辟一个相同大小的数组，每次把大于元素的数字放到尾巴处，小于的放到头处，这样时间复杂度是n，但是需要n的空间复杂度，考虑如何不使用多余的空间。
这种问题我们都可以考虑双工作指针的方法，所谓双工作指针，是说用两个指针把数组分成待处理，已处理两部分，已处理部分又分为两种性质，当然这个性质是很宽泛的，不应该死板的理解。类似的问题如荷兰国旗问题，Two Sum问题等。
快排里用left和right两个指针，其中left指向一个小于等于val的元素，并且left左边的位置都小于我们需要比较的元素。right指针右边的元素都大于我们要比较的元素。我们把left设置到数组开头，right设置到数组的尾部，可以满足上述性质，当left和right相遇的时候自然就结束了，在工作过程中要保证上述性质不变。

最短路径算法总结

Posted on 2017-03-09   |  

一、最短路径问题的定义

对于一个图，任意取两个点做起点和终点，可以规划出n条起点到终点的路线，不兜圈子不重复经过某一点的路线叫做一个路径（path），如果图中的边有权重，那么这些路径也就有了长度。

最短路径是从起点到终点长度最短的路径，当图没有负边的时候，这个问题是一个NP 完全的问题，最长路径问题类似，当没有正边是NP完全问题。

最短路径有一个很好的性质：

每一条最短路径，都是由其他最短路径延伸得到，也就是说对一个最短路径进行首尾切割得到的中间一段仍然是最短路径。

从一个起点出发，到图中所有点的最短路径构成一个最短路径树（可能不止一个）。

一个起点和终点间所有最短路径组成一个最短路径图，只有一个，他是一个有向無環圖（Directed Acyclic Graph, DAG）。

最短路径算法中一个经常用到的概念是松弛（relaxation），如下图所示[1]。

最短路径问题可以分为三类：

  1. Point to Point shorest path

  2. Single source shorest path

  3. All pairs shorest path

二、单源最短路径问题

1. Label Setting 算法

所谓label就是从起点到所有终点的距离值，label setting就是算法每次外循环set一个终点的值作为最后的结果。要怎么做呢？

定义w[a][b]为a顶点到b顶点的距离，也就是边的权重，定义d[a]为起点到a的距离，除了起点本身，其他都设为0。

  1. 将起点加到最短路径树上

  2. 重复V-1次，直到所有顶点加到最短路径树上

    a. 找一个不在最短路径树并且离起点最近的点，方法是找一个已经在最短路径上的点a以及一个不在最短路径点上的b，使得w[a][b] + d[a]最小。

    b. 将b的最短路径加到d[b]上

    c. 将b点连同ab边加到最短路径树上。

这里需要证明的一点是，所有与目前最短路径树连接的，与起点最近的一个顶点可以被加入到最短路径树上。

简单说一下我的理解：在每次循环开始时，我们已经有一个最短路径树T。按照上面算法第2步找到的顶点p一定是所有与这个树连接的顶点集合P中距离起点s最短的点，那么所有P中其他点到起点的距离都会长于p，因为没有负边，那么所有不在P和T中顶点到起点的距离更不可能比p到起点的距离短，所以这一步不会改变T是一个最短路径树的事实。

我们直接实现Label Setting算法，假如顶点数量是n，时间复杂度为O(n^3)。之后的Dijkstra算法对这个问题有改进。

2. Dijkstra算法

在Label Setting算法中，我们要用穷举的方式找到一个最近的点。实际上我们做了很多重复的计算，我们可能对某些顶点到起点的距离计算了很多遍。自然的，我们考虑用一个表来保存一些目前已经知道的“最短”距离，这个距离只是当前最短的，当对最短路径树加入新的顶点后，我们需要对这个表进行更新，也就是做一些“松弛”。

求和最大的子矩阵

Posted on 2017-03-01   |  

在微软面试的时候被问道一个题目，给一个m乘n的矩阵，矩阵内为一些值，现在求一个所有元素和最大的子矩阵。如矩阵
4 2 5
3 -4 5
2 1 -10
元素和最大的子矩阵为：
4 2 5
3 -4 5

首先考虑矩阵为一维的情况，我们的矩阵现在是一个数组$A = { a_1 , a_2, … ,a_n }$，现在要输出的是最大的子数组$a_i ~ a_j$。我们设置一个滑动窗口，窗口的左端x和右端y初始都设为下标为零处，我们再使用一个值res标记目前出现的，和最大的窗口值。然后我们从零开始迭代滑动窗口的右侧下标y。

  1. 当$a_x + .. + a_y = sum> 0$ ，取$res = max(res, sum)$，$y = y + 1$;
  2. 当$a_x + .. + a_y = sum <= 0$, $x = y + 1; y = y + 1$
  3. 当 y > n时，结束循环，最大值为res。
    在上面这个过程中，可以很方便的将窗口左右端也保存，这样就可以找到最大子数组的具体位置了。这个算法的时间复杂度为$O(n)$。

将一维情况推到二维矩阵$A = {a_{11}, a_12, …, a_1m, …., a_n1, a_n2, … , a_nm}$，如果使用暴力法，显然算法复杂度高达$O(n^4)$。
我们考虑利用一维的结果，首先我们计算一下所有的$r_i ~ r_j$（$1 <= i, j <= n$ )行的矩阵和，其结果为一个数列$B_ij = {b_1, b_2, …,b_m}$, 其中$b_k = a_ik + a_{(i+1)k} + … +a_jk$。获得$Bij$后，我们在上面执行一维的算法，找出最大的子数列$B\{ij}[x ~y]$及其值$res{ij}$这时候我们就找到了以$i j$为行边界条件下元素和最大的一个子矩阵，与其值$res\{ij}$，我们只需要遍历所有$i j$的组合，就可以找到全局和最大的子矩阵与其值$res$了。
在计算$B_ij$的时候，我们利用之前的计算结果即可。

Blur和Oasis是同一时期的两个不相上下的乐队，可能Blur比Oasis“有出息”一点，曲风更多变，又不像Radiohead那样阴郁，弱点在那种优美的旋律少了点，但是也不是没有，tender， you are so great就很好听，还有新专辑里的ong ong。抛却掉商业上的包装，blur的音乐就是那种各个方面都很好的摇滚乐，优质生，不脏，一上来你可能觉得不是那么好听，但是只要稍稍进入点状态就感觉他们确实很厉害。推荐他们的同名专辑《blur》。

The stone roses比之前几个乐队要早一点，说影响了后来的乐队一点也不夸张。一开始我是听惯了Oasis和blur去听石玫瑰的，感觉好像不够爆炸，直到最近我听了Oasis的lost tape，感觉似曾相识，好像挺像石玫瑰的，然后我又去听了《The Stone Roses》，这次就真的感觉爆炸了，各方面的冲击，觉得特别好。你考虑到他们还是自己搞出来的，搞出自己的味道，以前从来没有的，就更觉得厉害。

当从这几张专辑里走出来，再去听一些模仿的，没特点的，不好听更谈不上爆炸的乐队，真的感觉他们差距挺大的。

断舍离

Posted on 2016-11-19   |  

知乎上看到一个问题，如何让自己快速的静下来？有人回答收拾房间，这正符合我。我不是一个整整齐齐的人，但是我也喜欢整理房间，当东西妥妥当当，仿佛我会忘记一些烦恼事，让自己继续前进下去，似乎我又掌控了我的人生。

今天把自己的书桌整理了一下，已经是乱的不成样子，但是我之前竟然没有察觉。整理他花了一个多小时，其间丢了很多东西，也算是一种断舍离吧。记得高中的时候每次整理完屋子，丢了一些东西后，那种“断舍离”的感觉会强烈一点，一次我说给同学听，同学表示不理解，为什么要连自己的小工艺品也丢了呢，我说这个东西寄托了我无聊时间太多的情绪，丢了他我心情很轻松。于是这样我就可以向前了，于是我更无拘无束了，但是隐隐约约又觉得有所失，后来我又不太爱丢东西了。

其实连青春都是留不住的。走在冬天的路上，路边的冰用力也踩不破，过了两天居然连水也没留下了。更不用说那些草本植物，一茬换一茬。大自然倾向于替换，我们也是要被替换的东西。所以把旧的替换成新的，这是再正常不过的事情了，这样整个体系才能正常运转。

在我整理桌子时，发现居然还留着新生报道的手册，顺便又回想了一下自己怎么来的，然后把它丢了。恩，问题来了，丢了这个东西为什么我脑子里还在想呢，就像你搬到了异国他乡身边没有一个东西关于你的亲人，为什么你每时每刻都会想到他们呢，你被关到了监狱里，除了一个衣服身边什么都没有，为什么你还是记挂着自己的老婆孩子呢？物质的环境是一种线索和提醒，比如一辆车提醒了我可以驾车去很多地方，一个舒服的床是接下来我有一个好的睡眠的线索，一个整洁的环境常常能提醒自己凡事安排妥当减轻未知的焦虑，等等。而这些提醒之所以成立，也在于我们心里已经有了这些概念，当物质的线索没了，那些真正的渴望的东西会在精神里找线索。所以整理屋子，实际上是扫除那些令人不快的线索，从这个角度看，也不完全是体力活了。

因此，断舍离，不是在丢东西，而是一种心理暗示，我很轻松，我丢掉了包袱，被丢了的东西产生的空缺作为这种心里暗示在现实中的线索。

据说乔布斯的屋子开始一直很空荡荡，因为他只选自己喜欢的东西进来，这算是一种禅境了。

Posted on 2016-11-04   |   In 算法   |  

Given a set of candidate numbers (C) and a target number (T), find all unique combinations in C where the candidate numbers sums to T.

The same repeated number may be chosen from C unlimited number of times.

Note:

All numbers (including target) will be positive integers.
The solution set must not contain duplicate combinations.

For example, given candidate set [2, 3, 6, 7] and target 7,
A solution set is:

[ [7], [2, 2, 3]]
